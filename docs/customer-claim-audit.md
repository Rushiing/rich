## 1. Rubric(codex 请逐项套同一把尺)

每条对客断言打四个维度:

### 维度①「证据」— 这个判断有多少验证背书
| 档 | 含义 |
|---|---|
| `E-R` | 确定性规则/数据(算出来的,非预测 —— 行业分位、止损线数学、持仓盈亏)。可信 by construction,不参与"准不准"的怀疑。 |
| `E0` | 不可验证(本质上没法记分 —— 自由文本 valid_window、reasoning) |
| `E1` | LLM 自评、从没测(actionable 之外的多数 key_table 字段) |
| `E2` | 已埋点、数据积累中、暂无定论(price_level_stats,6/23 P0 起) |
| `E3` | 测了但弱/未校准(置信度分档、次日看跌/看平) |
| `E4` | 测了且有真实 edge(买入命中/5日超额) |

### 维度②「一致性」— 跟同屏其它信号冲突吗
`C0` 无冲突 · `C1` 潜在误读 · `C2` 直接矛盾

### 维度③「价值」— 值不值得继续投入维护
`V0` 看个意思(低用低背书,候选砍/收起) · `V1` 支撑(留,低维护) · `V2` 核心强项(投入)

### 维度④「翻译」— 能不能给用户一句真话讲清它好在哪
`T-yes` 有可验证的卖点 · `T-hedge` 留但须标"AI观点/参考" · `T-no` 讲不出真卖点 → 降级/砍

### 动作映射
- `E4 + V2` → 留、投入、翻译成卖点(护城河)
- `E1/E3 且以"自信/精确"形态展示` → **修/降级**(装饰性自信,真金最危险)
- `E0 以精确形态展示` → 改标"参考",去掉假精度
- `C2` → 调和
- `E-R` → 留,可信

## 2. 第一遍打分 —— 按"该做什么"分桶(不是 107 行流水账)

实测数据锚点(2026-06-23):
- 买入 5日超额 **+6.07pp**、去重命中 **64.3%**(n=148)→ 真 alpha
- 卖出 5日超额 **−0.09pp**(纯 beta,无个股 alpha)
- 次日看跌命中 **39%**(说跌实际涨1.4%)、次日置信 高52%<低58%(反/平)
- 主置信度 n=655:买入高置信 d1 **73.9%**>中53.5%(弱可信、样本薄);**卖出高置信 d1 40%/d5 20% < 中50% < 低57%(反向陷阱)**

### 🟢 真本事(E4 · V2 —— 投入 + 翻译成卖点)
| 项 | 打分 | 依据 |
|---|---|---|
| AI 历史命中率(去重)+ 5日超额 | E4 C0 V2 T-yes | 买入 +6.07pp 超额、去重64% —— 剥了 beta 的真选股力。**这是护城河。** |
| 诚实方法论(按天去重 / 剥market beta / 淘汰公开) | E4 C0 V2 T-yes | 不是单条断言,是信任基础设施。本身就是卖点。 |
| 预选池生命周期 + 失效线 + 淘汰公开 | E-R C0 V2 T-yes | 确定性规则、可对账;"把失败也摆出来"。 |
| 同业可比(明确标"数据直出不经AI") | E-R C0 V1 T-yes | 范例级诚实标注,这个 pattern 该扩散到全站。 |

### 🔴 装饰性自信 / 危险(修 / 降级 —— 真金最暴露)
| 项 | 打分 | 实测真相 → 处理 |
|---|---|---|
| **卖出 的置信度** | E3 C0 V0 T-no | **高置信卖出反而最不准(40%/20%)**。叠加卖出超额≈0。→ 卖出方向应**撤置信、或直接挂真实命中(50%上下,坦白"卖出信号弱")**。 |
| 次日走势预判「置信度」 | E3 C1 V0 T-no | 高52%<低58%,没校准。→ 撤置信,或换成真实命中(看涨63%/看跌39%)。 |
| 次日「看跌」预判本身 | E3 C0 V0 T-no | 命中39%、说跌实际涨 → 反向。→ 看跌单独重标/抑制,别当预测卖。 |
| valid_window 有效期 | E0 C0 V1 T-hedge | 自由文本、不可验证,却以精确"3交易日内"形态展示。→ 改标"参考时效",去假精度。 |
| risk_scores 综合评级 ⭐⭐⭐ | E1 C0 V0 T-no | LLM 自评8维、从没验证。→ 低价值,候选降级/收起。 |
| 买入/止损/目标 价位 | E2 C0 V1 T-hedge | P0 刚埋点、未验证。→ 继续展示(有用锚点)但**别加假精度**;等 price-level-stats 出数再校准。 |

### 🟡 AI 观点(留,标注为观点即可 —— 低风险)
deep_analysis、company_tag、one_line_reason、scenario 三档、red_flags、板块/个股推荐理由。
打分多为 `E1 C0 V1 T-hedge`:作为"AI 怎么看"是合理的,只要读起来是观点不是事实(页脚免责已部分覆盖)。**建议:统一一个"AI 观点"的视觉信号,跟 E4 的"实测背书"区分开。**

### ⚪ 规则/数据(E-R —— 可信,留)
行业水位/PE分位、止损线数学、持仓盈亏/安全垫、价格偏离警告(5%)、已过期(>4h)、池状态定义、大盘指数。by construction 可信,保留。

### 🔵 一致性冲突(调和)
| 项 | 状态 |
|---|---|
| 可推荐 vs 观望 同屏 | C2 → **已上过渡提示(6/23)** ✓ |
| data_completeness「输入完整度」被误读为 AI 可信度 | C1 → 需澄清:完整度 = 喂了多少料,≠ 判断有多准。改文案或明确二者关系。 |
| 主「置信度」买/卖不对称却同一个 UI | C1 → 买入高置信弱可信、卖出高置信反向,但前端同一个高/中/低 chip。**至少卖出方向要差异化处理。** |

## 3. keep / invest / cut(回答"哪些值得继续投入")
- **投入(护城河)**:命中率+超额闭环、预选池生命周期、同业可比、持仓对照、P0 价位记分(养着等数)。
