# RICH 三线解耦原则(买入 / 卖出 / 选股)

> 2026-06-25 定。北极星级原则,开任何买/卖/选股迭代前先对一眼这页。
> 本文只钉**原则与边界**,不写实现。

## 一句话

**买入、卖出、选股 —— 表面都是高频操作,底层逻辑会长成三件完全不相关的事。
幼苗期就分开,让每条线长自己的数据血脉、自己的记分、自己的战绩 —— 谁也不拖谁、
谁也不借谁的信用。**

## 为什么这三条注定分叉(三个不同的灵魂)

| 线 | 它在问什么 | 成功指标 | 用户时刻 |
|---|---|---|---|
| **选股** | 哪儿有 alpha 值得进视野? | 入池票后续超额 | 发现 / 拓展 |
| **买入** | 这只、现在、什么价,值不值得进? | 入场 edge(现 **+5pp**,复权安全口径) | 决策买点 |
| **卖出** | 我**手里**这只,当前状态是不是风险? | 避免回撤(独立攒,目前 ≈0,无战绩) | 持仓看护 |

输入不同、判据不同、验证不同、触达时刻不同。硬塞进**一个 `actionable` 枚举 /
一条解析管线**,等于三个灵魂挤一具身体 —— 选股被买入口径绑、卖出沦为买入残值。

## 边界规则(幼苗期)

1. **买入是已验证的幼苗,独立成长,不加第二消费者。** +5pp 还嫩,别让卖出/选股
   逻辑反向耦合它、把它当原料消费。改买入只为做强买入本身。
2. **卖出 = 纯「当前状态」风险信号,不引用任何过去时间切片。**
   - 解析是连续流(每天 N 档),**没有一个离散的「当时买入逻辑」可锚** —— 所以
     「我们当初建议买入的逻辑断了」这种叙事**不成立、不用**。
   - 卖出只对**当下可观测状态**负责(资金/技术/失效线等客观触发)。
   - **仓位语境(盈/平/亏)不只是措辞,是决定「动作」本身的结构性输入**:同一个风险
     信号,大赚→护利、小赚小亏→观察/轻减、大亏→护本/止损、未持有→只提示不催卖,
     动作各不相同。粗档由 ③ 漏斗已提供,够用(不需要精确成本价)。表述层面再走
     护利 / 护本,不踩处置效应、不说止盈/止损。
   - **不借买入的 +5pp 信用**(买 edge ≠ 卖 edge,借了是假的)。自攒自己的战绩
     (exit/hold/baseline 口径),攒够再对客亮「有效」。
   - 最大用武之地:用户**存量持仓里 RICH 从没推荐过的票**(自己买的、套着的)——
     买入轴够不着,独立风险轴才能看护。
3. **选股独立做强,服务「发现」,不被买入的逐票口径绑死。**
4. **跨线不借信用、不共记分。** 每条线的「有效」只能用自己那条线的战绩证明。
5. **解耦引擎,融合表达。** 后端的数据 / 记分 / 战绩按三线**独立**;但用户在「我这只
   持仓**下一步怎么办**」那一刻,前台要把三线结论**重新融合成一个可执行答案**(用持仓 /
   盈亏档语境)—— 不能甩给用户三个互不相干的判断。**决策表达的融合 ≠ 后端引擎的耦合。**
   (codex 6/25 攻防修正:用户问的不是「这票客观该不该买/卖」,是「我持有,下一步怎么办」。)

## 对迭代管线的含义

- 每个迭代**锚定单一条线**,不跨线纠缠;跨线的耦合默认视为债。
- 三条各自有独立的:数据字段血脉、打分/验证口径、对客措辞、触达入口。
- 评估「下一步做哪条」时,按 **价值缺口 × 可达性 × 与用户真实行为的贴合度** 排序,
  而不是按「哪条看起来酷」。
